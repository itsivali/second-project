# second-project

Enterprise platform scaffold generated by the Master Platform Bootstrapper.

## Stack

- Organization: itsivali
- Team: second-project
- Frontend: Next.js
- Gateway: Kong Gateway
- Cloud scaffold: Azure AKS
- Databases: PostgreSQL, Redis, Elasticsearch

## Services

- backend: NestJS (TypeScript)
- frontend: Go Fiber
- api-gateway: Python FastAPI

## Development

Use the Nix command with a space:

```bash
cp .env.example .env
nix develop
docker compose up --build
```

If you type `nix-develop`, your shell will normally report `command not found`
because the real Nix command is `nix develop`. This scaffold also includes a
compatibility helper:

```bash
./scripts/nix-develop
```

If `nix develop` itself is not found, install Nix on the host or use the
generated Dev Container. The Dev Container already installs the language,
Docker, Kubernetes, Helm, Terraform, and GitHub CLI toolchains through
devcontainer features.

Gateway endpoints:

- Nginx: https://localhost:8443
- Kong: http://localhost:8000
- APISIX: http://localhost:9080

Security service:

```bash
curl -s http://localhost:4000/health
curl -s -X POST http://localhost:4000/auth/token \
  -H 'content-type: application/json' \
  -d '{"subject":"admin","roles":["admin"]}'
```

## GitOps

The GitHub Actions workflow builds all Docker images and runs Trivy scans. The
Nix shell alias `deploy` triggers the workflow manually with `gh workflow run`.
